        300 +  ## Bash
        301 +  
        302 +  Bash (Bourne Again Shell) is the default command-line shell on macOS and most Linux distributions, providing a 
            + text-based interface for executing commands, running scripts, and automating workflows. While Windows uses 
            + PowerShell by default, Windows Subsystem for Linux (WSL) provides access to Bash on Windows systems.
        303 +  
        304 +  Understanding Bash is essential for intelligent textbook development because the MkDocs build system, Python 
            + script execution, Git version control, and deployment automation all rely on command-line operations.
        305 +  
        306 +  ### Shell vs. Terminal vs. Bash
        307 +  
        308 +  These terms are often used interchangeably but have distinct meanings:
        309 +  
        310 +  - **Terminal:** The application that provides a text interface (e.g., Terminal.app on macOS, Windows Terminal)
        311 +  - **Shell:** The program that interprets commands (e.g., Bash, Zsh, Fish, PowerShell)
        312 +  - **Bash:** A specific shell implementation, currently the most widely used on Unix-like systems
        313 +  
        314 +  When you open the integrated terminal in VS Code, you're opening a terminal application that runs a shell 
            + (typically Bash or Zsh on macOS/Linux, PowerShell on Windows).
        315 +  
        316 +  ### Bash Command Structure
        317 +  
        318 +  Bash commands follow a consistent structure:
        319 +  
        320 +  ```bash
        321 +  command [options] [arguments]
        322 +  ```
        323 +  
        324 +  For example, the command `ls -la /docs/chapters` breaks down as:
        325 +  
        326 +  - **Command:** `ls` (list directory contents)
        327 +  - **Options:** `-la` (long format, show hidden files)
        328 +  - **Arguments:** `/docs/chapters` (directory to list)
        329 +  
        330 +  Options typically start with `-` (single dash) for short options or `--` (double dash) for long options. 
            + Multiple short options can be combined: `-l -a` is equivalent to `-la`.
        331 +  
        332 +  ### Essential Bash Commands for Textbook Development
        333 +  
        334 +  The following commands are used frequently in intelligent textbook workflows:
        335 +  
        336 +  | Command | Purpose | Example |
        337 +  |---------|---------|---------|
        338 +  | `pwd` | Print working directory | `pwd` ‚Üí `/Users/username/textbook-project` |
        339 +  | `ls` | List directory contents | `ls -la docs/chapters` |
        340 +  | `cd` | Change directory | `cd docs/chapters/01-intro` |
        341 +  | `mkdir` | Create directory | `mkdir docs/sims/new-microsim` |
        342 +  | `touch` | Create empty file | `touch docs/chapters/05-graphs/index.md` |
        343 +  | `cp` | Copy files | `cp template.md chapter-03.md` |
        344 +  | `mv` | Move/rename files | `mv old-name.md new-name.md` |
        345 +  | `rm` | Remove files | `rm docs/chapters/draft.md` |
        346 +  | `cat` | Display file contents | `cat mkdocs.yml` |
        347 +  | `grep` | Search text | `grep "learning graph" docs/**/*.md` |
        348 +  | `chmod` | Change file permissions | `chmod +x scripts/install-skills.sh` |
        349 +  | `ln` | Create symbolic link | `ln -s ~/.claude/skills/glossary-generator ./` |
        350 +  
        351 +  ### Bash Environment and Variables
        352 +  
        353 +  Bash maintains environment variables that configure shell behavior and store system information. Common 
            + variables include:
        354 +  
        355 +  - `$HOME`: User's home directory (e.g., `/Users/username`)
        356 +  - `$PATH`: Directories searched for executable commands
        357 +  - `$PWD`: Current working directory
        358 +  - `$USER`: Current username
        359 +  
        360 +  You can display variable values using `echo`:
        361 +  
        362 +  ```bash
        363 +  echo $HOME      # /Users/username
        364 +  echo $PATH      # /usr/local/bin:/usr/bin:/bin
        365 +  echo $PWD       # /Users/username/textbook-project
        366 +  ```
        367 +  
        368 +  ### Command Chaining and Redirection
        369 +  
        370 +  Bash allows combining multiple commands using operators:
        371 +  
        372 +  - **Sequential execution (`;`):** Run commands one after another regardless of success
        373 +    ```bash
        374 +    cd docs/learning-graph; python analyze-graph.py learning-graph.csv quality-metrics.md
        375 +    ```
        376 +  
        377 +  - **Conditional execution (`&&`):** Run second command only if first succeeds
        378 +    ```bash
        379 +    mkdocs build --strict && mkdocs gh-deploy
        380 +    ```
        381 +  
        382 +  - **Output redirection (`>`):** Save command output to file
        383 +    ```bash
        384 +    python analyze-graph.py learning-graph.csv > quality-report.txt
        385 +    ```
        386 +  
        387 +  - **Append to file (`>>`):** Add command output to end of existing file
        388 +    ```bash
        389 +    echo "Quality check completed" >> build-log.txt
        390 +    ```
        391 +  
        392 +  - **Pipe (`|`):** Send output of one command as input to another
        393 +    ```bash
        394 +    ls -la | grep ".md"     # List only markdown files
        395 +    ```

        399 +  Efficient directory navigation is fundamental to command-line workflows, enabling quick access to chapter files,

        400 +  
        401 +  ### Understanding File Paths
        402 +  
        403 +  File paths specify the location of files and directories in the filesystem hierarchy. There are two types of 
            + paths:
        404 +  
        405 +  - **Absolute paths:** Start from the root directory (`/` on Unix, `C:\` on Windows)
        406 +    ```bash

        408 +    ```

        410 +  - **Relative paths:** Start from the current working directory
        411 +    ```bash
        412 +    # If current directory is /Users/username/Documents/textbook-project

        414 +    ```
        415 +  
        416 +  Special directory references:
        417 +  
        418 +  - `.` (single dot): Current directory
        419 +  - `..` (double dot): Parent directory
        420 +  - `~` (tilde): User's home directory
        421 +  - `-` (dash): Previous working directory
        422 +  
        423 +  ### Navigating the Filesystem
        424 +  
        425 +  The `cd` (change directory) command moves between directories:
        426 +  
        427 +  ```bash
        428 +  # Navigate to home directory

        430 +  
        431 +  # Navigate to specific project directory
        432 +  cd ~/Documents/textbook-project
        433 +  
        434 +  # Navigate to subdirectory (relative path)
        435 +  cd docs/chapters
        436 +  
        437 +  # Go up one level to parent directory
        438 +  cd ..
        439 +  
        440 +  # Go up two levels
        441 +  cd ../..
        442 +  
        443 +  # Return to previous directory
        444 +  cd -
        445 +  
        446 +  # Navigate to root directory
        447 +  cd /

        626 +  ## File Creation and Editing
        627 +  
        628 +  Command-line file creation and editing are essential skills for automating textbook workflows, especially when 
            + generating multiple files from templates or making bulk updates. While VS Code is the primary editor for content 
            + development, knowing command-line file operations enables scripting and automation.
        629 +  
        630 +  ### Creating Files
        631 +  
        632 +  The `touch` command creates empty files or updates the modification timestamp of existing files:
        633 +  
        634 +  ```bash
        635 +  # Create a new chapter index file
        636 +  touch docs/chapters/14-future-directions/index.md
        637 +  
        638 +  # Create multiple files at once
        639 +  touch docs/chapters/14-future-directions/{index.md,exercises.md,glossary.md}
        640 +  ```
        641 +  
        642 +  The `echo` command combined with output redirection creates files with initial content:
        643 +  
        644 +  ```bash
        645 +  # Create file with single line of content
        646 +  echo "# Chapter 14: Future Directions" > docs/chapters/14-future-directions/index.md
        647 +  
        648 +  # Append content to existing file
        649 +  echo "## Summary" >> docs/chapters/14-future-directions/index.md
        650 +  ```
        651 +  
        652 +  For multi-line content, use a here-document:
        653 +  
        654 +  ```bash
        655 +  cat << EOF > docs/chapters/14-future-directions/index.md
        656 +  # Chapter 14: Future Directions
        657 +  
        658 +  ## Summary
        659 +  
        660 +  This chapter explores emerging trends in AI-assisted education.

        662 +  ## Concepts Covered

        670 +  ### Editing Files
        671 +  
        672 +  While command-line text editors like `vim`, `nano`, and `emacs` are available, most intelligent textbook 
            + developers prefer editing in VS Code. However, simple text transformations can be performed using command-line 
            + tools:
        673 +  
        674 +  **`sed` (stream editor):** Perform find-and-replace operations
        675 +  
        676 +  ```bash
        677 +  # Replace all occurrences of "CMDB" with "Configuration Management Database"
        678 +  sed -i '' 's/CMDB/Configuration Management Database/g' docs/chapters/*/index.md
        679 +  
        680 +  # Add a line after a specific pattern
        681 +  sed -i '' '/## Summary/a\
        682 +  This chapter covers fundamental concepts.' docs/chapters/14-future-directions/index.md
        683 +  ```
        684 +  
        685 +  **`awk` (text processing):** Extract and transform structured text
        686 +  
        687 +  ```bash
        688 +  # Extract all level-2 headers from a file
        689 +  awk '/^## / {print $0}' docs/chapters/01-intro/index.md
        690 +  
        691 +  # Print only lines containing "learning graph"
        692 +  awk '/learning graph/ {print}' docs/chapters/*/index.md
        693 +  ```
        694 +  
        695 +  **`grep` (pattern matching):** Search for text patterns
        696 +  
        697 +  ```bash
        698 +  # Find all chapters mentioning "MicroSim"
        699 +  grep -r "MicroSim" docs/chapters/
        700 +  
        701 +  # Count occurrences of "learning graph" in all markdown files
        702 +  grep -r "learning graph" docs/ --include="*.md" | wc -l

        749 +  ## Shell Scripts
        750 +  
        751 +  Shell scripts are text files containing sequences of Bash commands that automate repetitive tasks. In 
            + intelligent textbook development, shell scripts are used to install Claude skills, generate content, validate 
            + quality, and deploy to production.
        752 +  
        753 +  ### Anatomy of a Shell Script
        754 +  
        755 +  A basic shell script has three components:
        756 +  
        757 +  1. **Shebang line:** Specifies the interpreter (always first line)
        758 +     ```bash
        759 +     #!/bin/bash
        760 +     ```
        761 +  
        762 +  2. **Comments:** Explain what the script does (start with `#`)
        763 +     ```bash
        764 +     # Install Claude skills to global skills directory
        765 +     ```
        766 +  
        767 +  3. **Commands:** The actual operations to perform
        768 +     ```bash
        769 +     ln -s $(pwd)/skills/* ~/.claude/skills/
        770 +     ```
        771 +  
        772 +  ### Example: Installing Claude Skills
        773 +  
        774 +  The `install-claude-skills.sh` script creates symbolic links from the project's skills directory to the global 
            + Claude skills directory:
        775 +  
        776 +  ```bash
        777 +  #!/bin/bash
        778 +  
        779 +  # Install Claude skills to global skills directory
        780 +  # This makes skills available to all Claude projects
        781 +  
        782 +  SKILLS_DIR="$HOME/.claude/skills"
        783 +  PROJECT_SKILLS="$(pwd)/skills"
        784 +  
        785 +  # Create skills directory if it doesn't exist
        786 +  mkdir -p "$SKILLS_DIR"
        787 +  
        788 +  # Link each skill to global directory
        789 +  for skill in "$PROJECT_SKILLS"/*; do
        790 +      skill_name=$(basename "$skill")
        791 +      echo "Installing skill: $skill_name"
        792 +      ln -sf "$skill" "$SKILLS_DIR/$skill_name"
        793 +  done
        794 +  
        795 +  echo "Skills installation complete!"
        796 +  ```
        797 +  
        798 +  ### Script Components Explained
        799 +  
        800 +  **Variables:**
        801 +  ```bash
        802 +  SKILLS_DIR="$HOME/.claude/skills"      # Directory where skills are installed
        803 +  PROJECT_SKILLS="$(pwd)/skills"         # Directory containing project skills
        804 +  ```
        805 +  
        806 +  **Command substitution:**
        807 +  ```bash
        808 +  skill_name=$(basename "$skill")        # Extracts filename from full path
        809 +  ```
        810 +  
        811 +  **For loops:**
        812 +  ```bash
        813 +  for skill in "$PROJECT_SKILLS"/*; do   # Iterate over each skill directory
        814 +      # Commands here execute for each skill
        815 +  done
        816 +  ```
        817 +  
        818 +  **Conditional creation:**
        819 +  ```bash
        820 +  mkdir -p "$SKILLS_DIR"                 # Create directory if it doesn't exist
        821 +  ```
        822 +  
        823 +  **Symbolic links:**
        824 +  ```bash
        825 +  ln -sf "$skill" "$SKILLS_DIR/$skill_name"    # -s = symbolic, -f = force (replace if exists)
        826 +  ```
        827 +  
        828 +  ### Script Best Practices
        829 +  
        830 +  Effective shell scripts follow these conventions:
        831 +  
        832 +  - **Start with shebang:** `#!/bin/bash` on line 1
        833 +  - **Use meaningful variable names:** `SKILLS_DIR` not `dir1`
        834 +  - **Quote variables:** `"$variable"` prevents word splitting
        835 +  - **Check for errors:** Use `set -e` to exit on any command failure
        836 +  - **Add help text:** Provide usage instructions when run with `-h` or `--help`
        837 +  - **Use functions:** Break complex scripts into reusable functions
        838 +  - **Validate inputs:** Check that required files/directories exist
        839 +  
        840 +  ### Example: Advanced Script with Error Handling
        841 +  
        842 +  ```bash
        843 +  #!/bin/bash
        844 +  set -e  # Exit on any error
        845 +  
        846 +  # Validate learning graph quality before deployment
        847 +  
        848 +  LEARNING_GRAPH_CSV="docs/learning-graph/learning-graph.csv"
        849 +  QUALITY_THRESHOLD=70
        850 +  
        851 +  # Check that learning graph file exists

        884 +  ## Script Execution Permissions
        885 +  
        886 +  Unix-like systems (macOS, Linux) use a permission system to control who can read, write, or execute files. 
            + Before a shell script can be run, it must have execute permissions set.
        887 +  
        888 +  ### Understanding File Permissions
        889 +  
        890 +  File permissions are displayed by `ls -l`:
        891 +  
        892 +  ```bash
        893 +  $ ls -l scripts/install-claude-skills.sh
        894 +  -rwxr-xr-x  1 username  staff  512 Jan 15 10:30 install-claude-skills.sh
        895 +  ```
        896 +  
        897 +  The permission string `-rwxr-xr-x` breaks down as:
        898 +  
        899 +  - **File type:** `-` (regular file), `d` (directory), `l` (symbolic link)
        900 +  - **Owner permissions:** `rwx` (read, write, execute)
        901 +  - **Group permissions:** `r-x` (read, execute, no write)
        902 +  - **Other permissions:** `r-x` (read, execute, no write)
        903 +  
        904 +  ### Permission Notation
        905 +  
        906 +  Permissions can be represented in two formats:
        907 +  
        908 +  **Symbolic notation:**
        909 +  ```
        910 +  r = read (4)
        911 +  w = write (2)
        912 +  x = execute (1)
        913 +  ```
        914 +  
        915 +  **Numeric notation (octal):**
        916 +  ```
        917 +  rwx = 4+2+1 = 7
        918 +  r-x = 4+0+1 = 5
        919 +  r-- = 4+0+0 = 4
        920 +  ```
        921 +  
        922 +  Common permission combinations:
        923 +  
        924 +  | Octal | Symbolic | Meaning |
        925 +  |-------|----------|---------|
        926 +  | 755 | -rwxr-xr-x | Owner can read/write/execute, others can read/execute |
        927 +  | 644 | -rw-r--r-- | Owner can read/write, others can read only |
        928 +  | 700 | -rwx------ | Owner can read/write/execute, others have no access |
        929 +  | 775 | -rwxrwxr-x | Owner and group can read/write/execute, others can read/execute |
        930 +  
        931 +  ### Making Scripts Executable
        932 +  
        933 +  To make a script executable, use the `chmod` command:
        934 +  
        935 +  ```bash
        936 +  # Add execute permission for owner
        937 +  chmod +x scripts/install-claude-skills.sh
        938 +  
        939 +  # Add execute permission for everyone
        940 +  chmod a+x scripts/install-claude-skills.sh
        941 +  
        942 +  # Set specific permissions using numeric notation
        943 +  chmod 755 scripts/install-claude-skills.sh
        944 +  ```
        945 +  
        946 +  After setting execute permissions, the script can be run directly:
        947 +  
        948 +  ```bash
        949 +  # Run with full path
        950 +  ./scripts/install-claude-skills.sh
        951 +  
        952 +  # Run with relative path
        953 +  cd scripts
        954 +  ./install-claude-skills.sh
        955 +  
        956 +  # Run from anywhere if in PATH
        957 +  install-claude-skills.sh
        958 +  ```

       1068 +  ## Symlink Creation
       1069 +  
       1070 +  Symbolic links (symlinks) are special files that act as pointers to other files or directories, enabling 
            + multiple paths to access the same content. In intelligent textbook development, symlinks are used to install 
            + Claude skills globally while maintaining the skills in the project repository.
       1071 +  
       1072 +  ### Why Use Symlinks for Skills?
       1073 +  
       1074 +  Claude Code looks for skills in `~/.claude/skills/` by default. Without symlinks, you would need to:
       1075 +  
       1076 +  - Copy skill files to `~/.claude/skills/` every time they're updated
       1077 +  - Maintain duplicate copies in each project
       1078 +  - Manually synchronize changes across projects
       1079 +  
       1080 +  Symlinks solve this by creating a reference in `~/.claude/skills/` that points to the original skill files in 
            + the project repository. When the original files are updated, the changes are immediately reflected in all 
            + projects using that symlink.
       1081 +  
       1082 +  ### Creating Symlinks
       1083 +  
       1084 +  The `ln` command creates symbolic links:
       1085 +  
       1086 +  ```bash
       1087 +  # Syntax
       1088 +  ln -s /path/to/original /path/to/link
       1089 +  
       1090 +  # Example: Link a single skill
       1091 +  ln -s ~/Documents/textbook-project/skills/glossary-generator ~/.claude/skills/glossary-generator
       1092 +  
       1093 +  # Example: Link all skills in a directory
       1094 +  ln -s ~/Documents/textbook-project/skills/* ~/.claude/skills/
       1095 +  ```
       1096 +  
       1097 +  **Flags:**
       1098 +  - `-s`: Create symbolic link (not a hard link)
       1099 +  - `-f`: Force overwrite if link already exists
       1100 +  - `-n`: Don't dereference existing symlink (useful when updating)
       1101 +  
       1102 +  ### Symlinks vs. Copies
       1103 +  
       1104 +  Understanding the difference is crucial:
       1105 +  
       1106 +  | Operation | Copy | Symlink |
       1107 +  |-----------|------|---------|
       1108 +  | Storage | Duplicates content | Only stores pointer (~1KB) |
       1109 +  | Updates | Manual re-copy needed | Automatic (follows original) |
       1110 +  | Deletion | Independent files | Deleting symlink doesn't affect original |
       1111 +  | Portability | Works if original is deleted | Breaks if original is moved/deleted |
       1112 +  | Permissions | Uses copy's permissions | Uses original's permissions |
       1113 +  
       1114 +  ### Verifying Symlinks
       1115 +  
       1116 +  Use `ls -l` to see symlink targets:
       1117 +  
       1118 +  ```bash
       1119 +  $ ls -l ~/.claude/skills/glossary-generator
       1120 +  lrwxr-xr-x  1 username  staff  72 Jan 15 10:30 glossary-generator -> 
            + /Users/username/Documents/textbook-project/skills/glossary-generator
       1121 +  ```
       1122 +  
       1123 +  The `->` arrow indicates this is a symlink pointing to the target path.
       1124 +  
       1125 +  ### Managing Symlinks
       1126 +  
       1127 +  **List all symlinks in a directory:**
       1128 +  ```bash
       1129 +  find ~/.claude/skills/ -type l -ls
       1130 +  ```
       1131 +  
       1132 +  **Check if a symlink target exists:**
       1133 +  ```bash
       1134 +  test -e ~/.claude/skills/glossary-generator && echo "Target exists" || echo "Broken symlink"
       1135 +  ```
       1136 +  
       1137 +  **Remove a symlink:**
       1138 +  ```bash
       1139 +  rm ~/.claude/skills/glossary-generator     # Removes link only, original unaffected
       1140 +  ```
       1141 +  
       1142 +  **Update a symlink to point to a new target:**
       1143 +  ```bash
       1144 +  ln -sf /new/path/to/skill ~/.claude/skills/glossary-generator
       1145 +  ```

       1147 +  ### Project-Local vs. Global Skills
       1148 +  
       1149 +  Claude Code supports two skill installation strategies:
       1150 +  
       1151 +  **Global skills (`~/.claude/skills/`):**
       1152 +  - Available to all projects
       1153 +  - Ideal for stable, mature skills used across multiple textbooks
       1154 +  - Installed via `scripts/install-claude-skills.sh`
